# AIResQ Backend AWS Services

This document maps the backend AWS resources used by the AIResQ app to their code paths and AWS console locations.

## Stack Overview

| Item | Value |
| --- | --- |
| CDK stack | `AiresqNativeAwsStack` |
| Backend root | `backend/` |
| Infrastructure entry | `backend/bin/airesq-native-aws.ts` |
| Infrastructure stack | `backend/lib/airesq-native-aws-stack.ts` |
| Default AWS region | `ap-south-1` |
| Default environment | `demo` |
| Project tags | `Project=AIResQ`, `Environment=demo`, `ManagedBy=AWS-CDK` |

The mobile app consumes the deployed backend values from `mobile/.env` or `mobile/.env.example`.

## API Gateway Routes

### HTTP API: `ReportsHttpApi`

Base URL: `https://d0xdj11ndh.execute-api.ap-south-1.amazonaws.com`

| Method | Path | Lambda | Auth |
| --- | --- | --- | --- |
| `GET` | `/captcha` | `CaptchaFn` | Public |
| `POST` | `/reports` | `CreateReportFn` | Public |
| `GET` | `/reports/public` | `PublicReportsFn` | Public |
| `GET` | `/admin/reports` | `AdminReportsFn` | Cognito JWT |
| `PATCH` | `/admin/reports/{id}/status` | `UpdateStatusFn` | Cognito JWT |
| `DELETE` | `/admin/reports/{id}` | `DeleteReportFn` | Cognito JWT |

### WebSocket API: `LiveUpdatesWebSocket`

WebSocket URL: `wss://b12zz2ibqc.execute-api.ap-south-1.amazonaws.com/prod`

| Route | Lambda | Purpose |
| --- | --- | --- |
| `$connect` | `WebSocketConnectFn` | Stores live client connection IDs in DynamoDB |
| `$disconnect` | `WebSocketDisconnectFn` | Removes disconnected client IDs from DynamoDB |
| Management API | Used by `CreateReportFn`, `UpdateStatusFn`, `DeleteReportFn` | Broadcasts `new_submission`, `report_updated`, and `report_deleted` events |

## Lambda Functions

| Function | Source path | Main AWS dependencies |
| --- | --- | --- |
| `CaptchaFn` | `backend/lambdas/captcha/index.js` | Lambda environment `CAPTCHA_SECRET` |
| `CreateReportFn` | `backend/lambdas/reports/create.js` | DynamoDB reports table, S3 photos bucket, WebSocket Management API |
| `PublicReportsFn` | `backend/lambdas/reports/listPublic.js` | DynamoDB reports table |
| `AdminReportsFn` | `backend/lambdas/reports/listAdmin.js` | DynamoDB reports table, S3 signed photo URLs |
| `UpdateStatusFn` | `backend/lambdas/reports/updateStatus.js` | DynamoDB reports table, S3 signed photo URLs, WebSocket Management API |
| `DeleteReportFn` | `backend/lambdas/reports/delete.js` | DynamoDB reports table, S3 photos bucket, WebSocket Management API |
| `WebSocketConnectFn` | `backend/lambdas/websocket/connect.js` | DynamoDB connections table |
| `WebSocketDisconnectFn` | `backend/lambdas/websocket/disconnect.js` | DynamoDB connections table |
| Shared helper | `backend/lambdas/shared/http.js` | DynamoDB client, S3 client, S3 presigner, API Gateway Management API client |

All Lambda functions run on Node.js 20 with 512 MB memory and a 30 second timeout.

`CreateReportFn` validates Cloudflare Turnstile tokens when the frontend sends `captcha.provider=turnstile`. The Lambda receives `TURNSTILE_PRIVATE_KEY` from CDK context, deployment environment, or local `mobile/.env` fallback.

## Notes

- Some physical resource names are generated by CloudFormation because the CDK stack does not set explicit names.
- The deployment region comes from `CDK_DEFAULT_REGION` or falls back to `ap-south-1`.
- Billing tools such as AWS Budgets and Cost Anomaly Detection appear in the Billing and Cost Management console rather than a regional service console.
- Activate the `Project` and `Environment` cost allocation tags in AWS Billing so spend reports group correctly.
